# Примеры списков в Markdown

## Маркированные списки

### Символ \* (звездочка)

- Первый пункт
- Второй пункт
- Третий пункт

### Символ - (дефис)

- Пункт один
- Пункт два
- Пункт три

### Символ + (плюс)

- Пункт A
- Пункт B
- Пункт C

---

## Нумерованные списки

1. Первый пункт
2. Второй пункт
3. Третий пункт
4. Четвертый пункт

---

## Вложенные списки

### Пример 1: Маркированный вложенный список

- Пункт 1
  - Подпункт 1.1
  - Подпункт 1.2
    - Подподпункт 1.2.1
    - Подподпункт 1.2.2
- Пункт 2
  - Подпункт 2.1
- Пункт 3

### Пример 2: Нумерованный вложенный список

1. Первый пункт
   1. Подпункт 1.1
   2. Подпункт 1.2
      1. Подподпункт 1.2.1
      2. Подподпункт 1.2.2
2. Второй пункт
   1. Подпункт 2.1
   2. Подпункт 2.2
3. Третий пункт

### Как создать репозиторий

#### Шаг 1: Подготовка структуры

Создать папку проекта со следующей структурой:

- Файл `README.md` для описания проекта
- Папка `src/` для хранения исходного кода
- Папка `docs/` для документации
- Файл `.gitignore` для исключения ненужных файлов

#### Шаг 2: Настройка репозитория

1. Инициализировать Git-репозиторий
   git init

2. Добавить все файлы в staging area
   git add .

3. Проверить статус изменений
   git status

4. Создать первый коммит
   git commit -m "Initial commit: базовая структура проекта"

#### Шаг 3: Работа с удалённым репозиторием

1. Настройка удаленного репозитория
   git remote add origin https://github.com/CR7MentalityN1/Lab3-MarkdownLaTeX_Lazimov.git
   git push -u origin master
